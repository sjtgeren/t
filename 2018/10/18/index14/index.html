九. 构造方法
构造器：根据用户的需求传入数据,
然后根据传入数据进行属性的初始化。
作用：为属性进行初始化
1）方法名和类名相同
2）没有返回值类型
3）构造的重载(根据不同的情况调用不同的构造器进行相应的初始化)
调用本类的构造器:this(参数列表)
4)系统会提供一个无参的构造器，
但是如果本类实现了其他的构造器，系统提供的无参构造器失效。
public class Test{
public Test(){}

一：继承
好处：代码复用，简化编程
代码复用用在哪些地方：
1)复用父类的属性
2)复用父类的方法(直接调用父类的方法)
3)重写父类的方法
实现：class 子类 extends 父类
单继承
关系：子类 is a 父类 或者 子类is a like 父类
class Student extends Person
student is a Person
Person is a Student（error）
子类继承父类哪些：
1）构造器不能继承
2）方法和属性全部继承

继承注意的问题：
1)创建一个子类对象必须先创建一个父类对象
2)子类的构造器会默认调用父类的无参构造器
super();
3)父类的无参构造器失效，
子类构造器就需要显式的调用父类存在的其他构造器
super(参数列表); 必须放在构造器的第一行
4)当一个子类继承了一个父类后，
父类中所有的字段和方法都被子类继承拥有，
子类可以任意的支配使用，
每个子类对象中都拥有了父类中的所有字段。
当构造一个子类的实例对象时，
该对象的实例变量包括了子类本身以及父类中的所有实例变量，
实例方法也包括了子类和父类中的所有实例方法。
子类构造器用来初始化子类中所有的实例变量，
而父类构造器super(实参)用来初始化父类中所有的实例变量。
那么在堆中为子类实例对象分配的内存区域中
包括了子类和父类中所有初始化后的实例变量。
5)在构造器同时出现this和super:不能够。

二：super代表父类对象
this代表当前对象
1)调用父类的属性(很少用)
super.name=name;
2)调用父类的构造器
super(参数列表);
3）调用父类的方法(子类中包含和父类相同的方法名)
super.方法名(参数列表);

二）扩展实验：

\1. 什么是构造器及其作用

构造器：根据用户的需求传入数据,

​ 然后根据传入数据进行属性的初始化。

​ 作用：为属性进行初始化

​ 1）方法名和类名相同

​ 2）没有返回值类型

​ 3）构造的重载(根据不同的情况调用不同的构造器进行相应的初始化)

​ 调用本类的构造器:this(参数列表)

​ 4)系统会提供一个无参的构造器，

但是如果本类实现了其他的构造器，系统提供的无参构造器失效。

\2. 方法重载有什么特点？定义几个重载方法和非重载方法？

​ 1.方法的重载必须在本类

​ 2.方法名字必须相同

​ 3.参数列表必须不同

​ 参数的数据类型不同||

​ 参数的顺序不同||

​ 参数的个数不同

​ 4.返回值可以不同

​ 5.抛出异常可以不同

​ 6.修饰符可以不同

3.this的作用？

this 代表当前对象

​ this用法:

​ a)this一般用在实例变量上

​ 用来区分实例变量和局部变量

​ 实例变量和局部变量相同名字的时候

​ this.name=name; 如果没有this，会执行就近原则，指代的就会是形参

​ b)this用在调用本类方法(不常用),系统会默认添加上去 this.setName(“”);

​ c)this调用本类构造器

\4. 子类继承了父类哪些东西，哪些东西不被继承？继承的关系？

​ a)子类继承父类的属性和方法，还可以扩展自己的属性和方法

​ b)子类不能够继承父类构造器
