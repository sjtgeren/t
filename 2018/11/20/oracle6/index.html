1.插入数据insert(只能插入一条数据)

注意：1)如果表后没有跟字段，

2)注意值和表中的字段的顺序和数据类型
3)不要违反完整性约束
4)先插入父表的数据，再插入子表的数据
5)插入空值—-null
6)如果表后有跟字段，
给表中指定的字段插入值，
其他的字段是null

3.更新已经存在的数据update
update 表名
set 字段名=值,字段名=值….
where 条件;
注意：a)不加where，修改表中所有数据
b)加where,修改表中满足where条件的数据
c)注意完整性约束
1)修改所有的gender为m,age为20

3)修改张三电话号码都为10010。
update s_stu
set phone=’10010’
where name=’张三’;
4.删除数据delete
delete [from] 表名
where 条件;
a)不加where,删除表中所有数据
b)不加where,删除表中满足where的数据
c)删除表中的数据，不会删除表结构
d)先删除子表的数据，再删除父类的数据
e)在创建表的时候，给外键加了
on delete cascade,就可以直接父类的数据
1)删除s_stu，name为李四的学生

2)删除s_stu所有数据

5.事务
a)事务是一个操作单元中的一系列操作，
这些操作是不可再分的原子操作，
这些原子操作要么同时成功，要么同时失败。
b)事务开始
1)打开SQLPLUS工具时，事务开始
2)前一个事务结束，新的事务开始
c)事务结束
1)commit,rollback
2)值DDL，DCL语句的会自动提交
3)exit会自定提交
4)非正常情况(系统错误,非常退出,系统崩溃)
rollback
d)事务特点：ACID
1)原子性：事务不可再分
2)一致性：事务操作前后的数据总量保持一致
3)隔离性：一个事务不能看到另外一个事务数据
4)持久性：数据要持久化的保存

e)DML没有自动提交，需要手动提交
insert delete update
网上购物(
生成一个订单(insert)
生成一个订单明细的数据(insert)
扣钱(update)
更新库存量(update)
commit;
)

1)针对字段的操作
a)添加一个字段

add (字段名 数据类型 默认值 非空约束,
…..);
注意：1)如果只添加一个字段，可以把()去掉

b)删除一个字段

c)修改一个字段:
修改数据类型，长度，默认值，非空约束

modify
(字段名 数据类型(长度) 默认值 非空约束);
注意：1)比如长度大于或者等于值的最大长度
或者没有数据。才修改成功
2)比如数据类型，没有数据的时候才
修改成功

2.针对约束
a)增加约束,不能增加非空约束
alter table 表名
add 取约束名字 约束类型(字段s);
b)删除约束
alter table 表名
drop constraint 约束名字;
////////////////
alter table 表名
drop primary key cascade;
删除主键约束，级联删除它的子表的外键约束

c)使约束生效
alter table 表名
enable constraint 约束名字 [cascade];
注意：会检查存在的数据是否违反约束
d)使约束失效
alter table 表名
disable constraint 约束名字 [cascade];

3.删除表
a)drop table 表名 [CASCADE CONSTRAINTS];
1)删除表中数据
2)删除表的结构
3)是DDL语句，自动提交，不能回滚
4)可以删除所有对象
b)delete from 表名;
1)删除表中数据
2)不会删除表的结构
3)是DML语句，可以回滚
4)删除数据的效率低，
将删除数据写入到日记文件
c)truncate 截断表
1)删除表中数据
2)不会删除表的结构
3)是DDL语句，自动提交，不能回滚
4)只能删除表，不能删除其他对象
5)删除数据的效率高

第十二章: 视图view
视图就是提取一张或者多张表的数据生成一个映射，操作视图可以达到操作原表的效果，方便数据的管理以及安全操作。

视图的作用:
1.隐藏表中的重要数据
2.代替一些比较长的sql语句
视图分为俩类:
简单视图:
视图所代表的sql中如果没有group by语句,没有组函数,查询的只有一张表,那么这样的视图就是简单视图.
复杂视图
视图所代表的sql中如果有group by语句,或者有组函数,或者查询的是多张表,那么这样的视图就是复杂视图.

简单视图和复杂视图的区别:
通过简单视图可以修改原来表中的数据,通过复杂视图是不能修改原来的数据的。
创建视图：
create or replace view 视图名字
as
sql语句

删除视图

第十三章: 索引(index)
索引的概念：
1. 类似书的目录结构
2. Oracle 的”索引”是一种对象，是与表关联的可选对象，能提高SQL查询语句的速度
3. 索引直接指向包含所查询值的行的位置，减少磁盘I/O
4. 索引和表是相互独立的物理结构
5. Oracle 自动使用并维护索引，插入、删除、更新表后，自动更新索引，所以说我们最多创建所以其他的都有数据库自动控制
索引的创建：
1.自动创建
当在表中指定了primary Key或者unique约束时会自动创建唯一值索引。
2.用户创建。
用户可以创建非唯一值索引以提高在访问数据时的效率。
语法：
create index 索引名
on 表名(列名);
例如：
create index emp_index
on s_emp(last_name);
删除索引:
drop index 索引名;
例如:
drop index emp_index;
创建成功后可以通过如下语句查看：
select index_name from user_indexes;
给某列创建索引的原则：
1.列经常作为where子句的限定条件或者作为连接条件
2.列包含的数据量很大，并且很多非空的值。
3.两个或者更多列频繁的组合在一起作为where的限定条件或者连接条件
4.列总是作为搜索条件
5.索引查出的数据量占2%~4%
6.索引不是越多越好，不是索引越多越能加速查找。
7.要建索引的表不经常进行修改操作

注意:
1.在表中的某一个合适的列加入上了索引,那么也只有在数据量很大的时候,才能有所体现出这个查询的效率.
2.索引一旦建立成功,那么之后这个索引就由系统来管理,我们自己是控制不了的.

索引的种类:
Single column 单行索引
Concatenated 多行索引
Unique 唯一索引
NonUnique 非唯一索引
索引结构分为:
B-tree(默认是这种结构)
适合大量的增、删、改（OLTP）；
不能用包含OR操作符的查询；
适合高基数的列（唯一值多）
典型的树状结构；
